09. Project: Building SPP-Twitter

In this final project, we will apply everything we've learned to build a functional **Twitter-style clone**. This application will support posting tweets, following users, and a real-time global timeline.

Step 1: Data Modeling (YAML)

We need two primary entities: Tweet and Follower.

Tweet Entity (etc/entities/tweet.yml)

table: tweets
id_field: id
attributes:
  user_id: int
  content: varchar(280)
  posted_at: timestamp
relations:
  - parent_entity: User
    parent_entity_field: user_id
    child_entity_field: id
    relation_type: ManyToOne

Follower Entity (etc/entities/follower.yml)

table: followers
id_field: id
attributes:
  follower_id: int
  following_id: int
  since: timestamp

Step 2: The Social Module

Create a module to encompass our social logic: Social.

php spp/spp.php make:module Social

Posting a Tweet (class.social.php)

namespace SPPMod\Social;

class Social extends \SPP\SPPObject {
    public static function postTweet($content) {
        $userId = \SPPMod\SPPAuth\SPPAuth::getCurrentUserId();
        
        $tweet = new \SPPMod\SPPEntity\SPPEntity('tweet');
        $tweet->set('user_id', $userId);
        $tweet->set('content', $content);
        $tweet->save();

        // Fire an event for real-time updates!
        \SPP\SPPEvent::fireEvent('onTweetPosted', ['tweet' => $tweet]);
    }
}

Step 3: Security & Profiles

We'll use SPPAuth to handle logins. Each student in our "V-Shiksha" portal from earlier can now have a social profile.

if (!\SPPMod\SPPAuth\SPPAuth::isLoggedIn()) {
    \SPPMod\SPPView\ViewPage::redirect('login');
}

Step 4: The UI & Timeline

Define the timeline route in etc/pages.yml:

pages:
  - name: timeline
    url: /timeline.php

timeline.php (src/timeline.php)

<div id="tweet-input">
    <form action="post_tweet.php" id="tweet_form">
        <textarea name="content" placeholder="What's happening?"></textarea>
        <button type="submit">Tweet</button>
    </form>
</div>

<div id="timeline-list">
    <?php
    $tweets = \SPPMod\SPPEntity\SPPEntity::loadAll('tweet', 'posted_at DESC');
    foreach ($tweets as $t) {
        echo "<div class='tweet'>" . $t->get('content') . "</div>";
    }
    ?>
</div>

Step 5: SPA Magic (FormAugmentor)

By defining our form in XML, FormAugmentor will instantly make the timeline dynamic.

<form name="tweet_form" action="/post_tweet.php">
    <controls>
        <control name="content" type="ViewInputTextarea" />
    </controls>
</form>

Now, when a user clicks "Tweet", the feed updates instantly without a page reload!

Step 6: Going Live (SPPLive)

To alert other users of new tweets in real-time, hook into the onTweetPosted event:

\SPP\SPPEvent::registerHandler('onTweetPosted', function($payload) {
    \SPPMod\SPPLive\SPPLive::broadcast('timeline_update', $payload['tweet']->toArray());
});
Bonus: AI Sentiment Analysis

Use SPPAI to automatically label toxic or positive tweets during the postTweet logic:

$sentiment = \SPPMod\SPPAI\SPPAI::analyzeSentiment($content);
$tweet->set('sentiment', $sentiment);
